# 客户池功能模块 PRD

## 1. 基本信息
- **项目名称：** 客户池（Customer Pool）
- **模块路径：** `mini-crm/customer-manage/customer-pool`
- **创建时间：** 2026-03-24
- **最后更新：** 2026-03-24
- **状态：** 需求已整理

## 2. 功能概述
客户池是一个客户资源管理模块，用于集中管理未分配或公海客户资源，支持客户信息的查看、分配、编辑和删除操作。通过筛选条件快速查找目标客户，并进行批量或单个客户的管理操作。

## 3. 用户故事

### US-1: 查看客户池列表
**作为** 销售经理或销售人员
**我想要** 能够查看客户池中的所有客户列表
**以便于** 了解当前可用的客户资源，并进行后续的客户分配和跟进

**验收标准：**
- 页面加载时自动显示客户池列表
- 支持分页显示
- 默认按客户分配时间倒序排列

### US-2: 筛选客户
**作为** 销售人员
**我想要** 能够通过多种条件筛选客户
**以便于** 快速找到符合特定条件的客户

**验收标准：**
- 支持按客户名称搜索
- 支持按手机号搜索
- 支持按客户来源筛选
- 支持按负责人筛选
- 筛选条件支持重置

### US-3: 查看客户详情
**作为** 销售人员
**我想要** 能够查看客户的详细信息
**以便于** 了解客户的完整信息

**验收标准：**
- 点击"查看"按钮打开详情弹窗
- 以只读方式显示所有客户信息
- 支持关闭弹窗

### US-4: 编辑客户信息
**作为** 销售人员
**我想要** 能够编辑客户的基本信息
**以便于** 更新客户资料

**验收标准：**
- 点击"编辑"按钮打开编辑弹窗
- 弹窗中显示客户的所有可编辑字段
- 支持保存修改
- 保存成功后提示并刷新列表

### US-5: 删除客户
**作为** 销售经理
**我想要** 能够删除无效或错误的客户记录
**以便于** 保持客户池数据的准确性

**验收标准：**
- 点击"删除"按钮弹出确认对话框
- 确认后删除客户记录
- 删除成功后提示并刷新列表

### US-6: 新增客户
**作为** 销售人员
**我想要** 能够手动添加新的客户到客户池
**以便于** 录入新获得的客户信息

**验收标准：**
- 点击"新增"按钮打开新增弹窗
- 填写必要信息后保存
- 保存成功后提示并刷新列表

## 4. 页面设计

### 4.1 页面结构

```
客户池页面
├── 页面标题区域
│   └── "客户池"
├── 筛选条件区域（表单）
│   ├── 客户名称输入框
│   ├── 手机号输入框
│   ├── 客户来源下拉选择
│   └── 负责人下拉选择
├── 操作工具栏
│   └── "新增"按钮
└── 数据表格区域
    ├── 序号列
    ├── 客户编码列
    ├── 客户名称列
    ├── 手机号列
    ├── 客户来源列（字典翻译）
    ├── 客户等级列（字典翻译）
    ├── 客户行业列
    ├── 负责人列
    ├── 客户分配时间列
    ├── 客户状态列（字典翻译）
    └── 操作列（查看、编辑、删除）
└── 分页区域
```

### 4.2 筛选条件字段

| 字段名称 | 字段类型 | 组件类型 | 说明 |
|---------|---------|---------|------|
| 客户名称 | String | Input | 支持模糊搜索 |
| 手机号 | String | Input | 支持模糊搜索 |
| 客户来源 | String | Select | 数据来源：minicrm_customer_source 字典 |
| 负责人 | Number | ApiSelect | 调用 getSimpleUserList 接口获取用户列表 |

### 4.3 表格列字段

| 列名 | 字段名 | 宽度 | 说明 |
|-----|-------|------|------|
| 序号 | seq | 80px | 固定左侧，居中对齐 |
| 客户编码 | code | 180px | 客户的唯一编码 |
| 客户名称 | name | 160px | 客户名称 |
| 手机号 | phone | 160px | 联系电话 |
| 客户来源 | customerSource | 160px | 字典翻译，minicrm_customer_source |
| 客户等级 | customerLevel | 160px | 字典翻译，minicrm_customer_level |
| 客户行业 | industry | 160px | 所属行业 |
| 负责人 | managerUser | 160px | 用户昵称 |
| 客户分配时间 | assignTime | 160px | 格式化显示 |
| 客户状态 | status | 160px | 字典翻译，minicrm_customer_status |
| 操作 | actions | 240px | 固定右侧，包含查看、编辑、删除 |

### 4.4 操作按钮

#### 页面级操作
- **新增**：打开新增弹窗，添加新客户

#### 行级操作
- **查看**：以只读模式打开详情弹窗
- **编辑**：打开编辑弹窗，修改客户信息
- **删除**：弹出确认对话框，确认后删除客户

## 5. 交互说明

### 5.1 筛选交互
1. 用户在筛选条件中输入或选择条件
2. 点击"查询"按钮（或回车）触发查询
3. 表格数据根据筛选条件刷新
4. 点击"重置"按钮清空所有筛选条件，恢复默认列表

### 5.2 表格交互
1. 列表支持全选/单选
2. 点击列头可进行排序
3. 操作按钮点击触发对应功能
4. 分页切换时自动刷新数据

### 5.3 弹窗交互
- **新增/编辑弹窗**：
  - 弹窗标题根据操作类型显示（新增/编辑客户）
  - 底部显示"确定"和"取消"按钮
  - 表单验证失败时显示错误提示
  - 保存成功后自动关闭弹窗并刷新列表

- **查看详情弹窗**：
  - 弹窗标题显示"查看客户"
  - 所有字段以纯文本形式展示
  - 底部只显示"关闭"按钮
  - 不支持修改操作

### 5.4 删除交互
1. 点击"删除"按钮
2. 弹出确认对话框："确定删除客户 [客户名称] 吗？"
3. 点击"确定"执行删除
4. 点击"取消"取消删除操作
5. 删除过程中显示加载提示
6. 删除成功后显示成功提示并刷新列表

## 6. 数据字典

### 6.1 客户来源（minicrm_customer_source）
- 建议值：线上推广、线下活动、客户介绍、自主开发、其他

### 6.2 客户等级（minicrm_customer_level）
- 建议值：A类（重点客户）、B类（潜力客户）、C类（普通客户）、D类（低价值客户）

### 6.3 客户状态（minicrm_customer_status）
- 建议值：待跟进、跟进中、已成交、已流失、已失效
