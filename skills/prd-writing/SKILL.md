# PRD 需求文档编写

## 输出模板

每个需求文档必须包含以下结构：

```markdown
# [功能名称] - 需求文档

## 1. 概述
- **功能名称：**
- **优先级：** P0/P1/P2/P3
- **目标用户：**
- **业务背景：**

## 2. 用户故事
| 编号 | 角色 | 行为 | 期望结果 |
|------|------|------|----------|
| US-01 | | | |

## 3. 页面列表
| 页面名 | 说明 | 入口 |
|--------|------|------|
| | | |

## 4. 交互说明
### [页面名]
- 正常流程：...
- 异常流程：...
- 边界条件：...

## 5. 数据字段
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

## 6. 非功能需求
- 性能要求：
- 兼容性：
- 安全性：

## 7. 验收标准
- [ ] ...
```

## 原则
- 需求模糊时主动提问，不要猜测
- 每个页面单独拆分一个 md 文件，方便下游 Agent 处理
- 交互说明必须覆盖正常、异常、边界三种场景
